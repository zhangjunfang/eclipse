使用说明

简单的介绍下如何使用该程序抓取分析http协议

1.客户端（如手机）

找到wifi网络设置项
设置代理为 手动
主机地址 ： {{.pproxy_host}}
主机端口 ： {{.pproxy_port}}
推荐android 用户使用类似 proxyDroid 这样的应用来管理多个代理

2.服务端-前端

进入首页的session列表查看所有http会话。
输入客户端的ip地址 或者http basic认证的用户名来过滤查看对应的会话信息。
所有的输入框都可以使用|来输入多个条件(或)。如user:表示过滤出用户为a或b的。
redo 功能可以很方便的让我们重发请求。

3.改写请求

即对request进行修改。
pproxy使用js代码来进行配置，大致如下 
function rewrite(req){
    //here you code
    return req
}
url : http://www.example.com/album/list?cid=126
req对象有如下一下属性：
schema : http
host : www.example.com
port : 80
path : /album/list
get: {cid:[123]}
post: {}
username : 
password : 
method: GET
form_get  : {add:function(k,v){},set:function(k,v){},get:function(k){},len:function(){}} 
form_post : {add:function(k,v){},set:function(k,v){},get:function(k){},len:function(){}}

host_addr: #修改该请求的host是使用，如 127.0.0.1:3218

#注意 get 和post的值是数组，如上cid参数
#form_get 用于更方便的操作  get参数对象
#form_post 用于更方便的操作 post参数对象

4.配置项说明

 客户端认证类型：
 authType:{
    0: 无认证
    1: http basic auth
    2: http basic,用户名不为空即可
 }
 
 response在何时进行存储：
 responseSave:{
    0: 所有
    1: request在session列表中出现过
 }
 
 会话查看限制条件：
 sessionView:{
 0 : 无限制，会话发送给所有人
 1 : 客户端ip或者http认证的用户名其中一个不为空
 }
